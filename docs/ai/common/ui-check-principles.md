# UI Check Principles

이 문서는 공용 UI 검수 기준이다.

## 원칙

- 기능 동작과 UX 안정성은 별도로 본다.
- 컴파일 성공은 UI 성공이 아니다.
- 화면은 실제 사용 흐름 기준으로 확인한다.
- 모바일과 데스크톱 주요 폭을 함께 고려한다.

## 확인 항목

- 텍스트가 잘리거나 겹치지 않는가
- 버튼과 입력의 높이, 줄바꿈, 정렬이 안정적인가
- 패널, 메뉴, 말풍선, 모달이 화면을 밀어내지 않는가
- 내부 스크롤과 외부 스크롤이 충돌하지 않는가
- 상태 변화에 따라 레이아웃이 흔들리지 않는가
- 확대, 축소, 재생, 선택 변경 중 요소가 사라지거나 높이가 접혔다 펼쳐지지 않는가
- 작업 흐름의 마지막 위치에서 저장, 삭제, 다음 이동 같은 완료 액션을 찾을 수 있는가
- 설명 문구가 액션 버튼에 밀려 너무 좁게 표시되지 않는가
- 사용자가 다음 행동을 이해할 수 있는가

## 보고 기준

- 기능은 되지만 어색한 부분을 별도로 적는다.
- 화면 흔들림, overflow, 모바일 문제는 risky로 승격할 수 있다.
- 가능한 경우 실제 화면 또는 스크린샷 기준으로 판단한다.
