# Review Principles

이 문서는 공용 리뷰 기준이다.

## 리뷰 대상

- 요청 범위 이탈
- 책임 경계 침범
- 중복 구현
- 기존 패턴 우회
- 상태 흐름 회귀
- 저장/삭제 위험
- 테스트 누락

## 원칙

- 리뷰어는 구현하지 않는다.
- 새 기능 제안보다 위험 탐지를 우선한다.
- 취향이 아니라 회귀 가능성과 유지보수 위험을 지적한다.
- 문제가 없으면 문제가 없다고 명시한다.
- 남은 리스크와 미검증 영역을 함께 적는다.

## 좋은 리뷰 출력

- 파일과 위치가 명확하다.
- 재현 조건 또는 실패 가능성이 설명된다.
- 심각도 순서로 정리된다.
- 해결 방향은 최소한으로 제안한다.
