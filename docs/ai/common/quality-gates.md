# Quality Gates

이 문서는 공용 완료 기준이다.

작업 완료는 기능 구현, 화면 사용성, 검증 결과를 분리해서 판단한다.

## 공통 완료 기준

- 요청한 기능이 실제 사용자 흐름에서 끝까지 수행되는가
- 사용자가 현재 화면의 목적, 다음 행동, 완료 행동을 이해할 수 있는가
- 저장, 삭제, 다음 이동, 취소 같은 종료 액션이 작업 흐름의 끝 지점에 있는가
- 빈 상태, 오류 상태, 리로드 상태, 데이터 전환 상태가 깨지지 않는가
- 큰 데이터, 긴 텍스트, 큰 이미지, 작은 이미지에서 레이아웃이 무너지지 않는가
- 기능 추가가 기존 사용자 흐름을 후퇴시키지 않았는가
- 공통으로 처리할 수 있는 UI나 상태 처리를 화면별 임시 구현으로 늘리지 않았는가

## UI 작업 완료 기준

- 주요 화면 폭에서 텍스트, 버튼, 입력, 상태 메시지가 겹치거나 잘리지 않는가
- 동적 크기 변화가 부모 영역 높이를 불필요하게 접었다 펼치지 않는가
- 확대, 축소, 재생, 선택 변경처럼 반복 조작하는 기능이 깜빡임이나 layout shift를 만들지 않는가
- 스크롤이 긴 화면에서 완료 액션을 다시 찾기 위해 사용자가 흐름을 역행하지 않아도 되는가
- 설명 문구가 버튼이나 액션 영역에 밀려 의미를 잃지 않는가
- 버튼 크기와 정렬이 주변 입력, select, 링크와 일관되는가

## 데이터 작업 완료 기준

- 새 데이터 생성, 저장, 삭제 후 목록과 미리보기가 같은 상태를 가리키는가
- 다른 대상이나 다른 데이터로 전환했을 때 이전 상태가 화면에 남지 않는가
- 저장 후 리로드해도 의도한 상태가 복원되는가
- 삭제 후 선택값, 미리보기, 상태 메시지가 안전한 기본 상태로 돌아가는가

## 완료 판정

다음 중 하나라도 확인하지 못했다면 완료 보고에 남은 리스크로 적는다.

- 관련 검증 명령
- 관련 화면 접근
- 콘솔 오류 여부
- 핵심 사용자 시나리오
- 리로드 또는 데이터 전환 시나리오
