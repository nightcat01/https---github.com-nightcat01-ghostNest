# Self A2A Failure Modes

이 문서는 하나의 AI 하네스가 역할을 나누어 수행할 때 생기는 고유한 실패 모드를 줄이기 위한 공통 기준이다.

## 목적

Self A2A는 역할 분리처럼 보이지만 실제 실행 주체가 같기 때문에 검토가 약해지거나, 오래된 전제가 반복되거나, 이전 판단을 스스로 정당화하는 문제가 생길 수 있다.

따라서 긴 작업, normal 이상 작업, 컨텍스트 재개 작업, reviewer/tester가 필요한 작업에서는 이 문서를 함께 확인한다.

## 주요 실패 모드

### Role Drift

역할이 섞여 Worker가 Reviewer 결론까지 대신 내리거나, Reviewer가 새 기능을 제안하고 구현 방향을 끌고 가는 현상이다.

대책:

- 각 단계 시작 전에 현재 역할과 금지 사항을 짧게 재확인한다.
- Reviewer는 구현하지 않고 문제 탐지만 한다.
- Tester는 검증 결과를 추정하지 않고 실행 여부와 결과를 분리해서 적는다.

### Consensus Rot

Planner, Worker, Reviewer가 서로 다른 관점처럼 보이지만 실제로는 앞 단계 결론을 그대로 반복하며 검토 강도가 떨어지는 현상이다.

대책:

- Reviewer는 최소 한 번 "이 변경이 틀렸다면 어디서 틀릴 수 있는가"를 확인한다.
- 리뷰 결과에는 근거 파일, 실행 결과, 관찰된 UI 중 하나 이상을 포함한다.
- 근거가 없으면 "확인하지 못함"으로 남기고 통과로 간주하지 않는다.

### Context Rot

오래된 대화, 이전 구현 의도, 이미 바뀐 파일 상태를 현재 사실처럼 취급하는 현상이다.

대책:

- 작업 재개, 긴 작업, 파일 변경 후에는 관련 파일을 다시 읽는다.
- 사용자 발언보다 현재 파일 상태가 우선 필요한 경우 실제 파일을 기준으로 판단한다.
- 이전 결론을 사용할 때는 현재 코드와 충돌하지 않는지 확인한다.

### Rubber-Stamp Review

자기 구현을 스스로 리뷰하면서 "문제 없어 보임"으로 지나가는 현상이다.

대책:

- Reviewer는 칭찬이나 요약보다 결함 탐색을 우선한다.
- 문제가 없을 때도 남은 테스트 공백이나 확인하지 못한 영역을 적는다.
- UI 작업은 코드만 보고 통과시키지 않고 가능한 경우 화면 결과를 확인한다.

### Scope Creep Normalization

작업이 커졌는데도 처음 위험도 그대로 진행하는 현상이다.

대책:

- 수정 파일 수, core/runtime 영향, 저장/삭제, 공통 UI 영향이 늘어나면 즉시 escalation 한다.
- 위험도가 올라가면 필요한 특화 역할을 추가한다.
- 원래 요청과 직접 관련 없는 리팩토링은 분리한다.

### Validation Theater

검증을 한 것처럼 보이지만 실제로는 빌드 통과만으로 UX, 데이터, 런타임 동작까지 통과했다고 간주하는 현상이다.

대책:

- 타입체크, 빌드, 브라우저 확인, API 확인, 데이터 보존 확인을 서로 다른 검증으로 구분한다.
- 실행하지 않은 검증은 이유와 잔여 리스크를 남긴다.
- "빌드 통과"는 화면 정상 동작의 증거로 쓰지 않는다.

## Rot Checkpoint

다음 상황에서는 짧게 rot checkpoint를 수행한다.

- 작업이 30분 이상 이어짐
- 컨텍스트 요약 또는 재개 후 다시 작업함
- 변경 파일이 처음 예상보다 늘어남
- 같은 문제를 두 번 이상 수정함
- Reviewer 또는 Tester 판단이 근거 없이 낙관적으로 느껴짐
- 사용자 피드백이 "아직 이상하다", "다시 확인해줘", "없어진 것 같다"에 가까움

확인 항목:

- 현재 판단이 실제 파일 상태에 근거하는가
- 이전 결론을 반복하고 있지는 않은가
- 반대 증거를 찾았는가
- 역할별 책임이 섞이지 않았는가
- 검증하지 않은 것을 통과로 말하고 있지는 않은가

## 결과 기록 형식

필요할 때 다음 항목을 짧게 남긴다.

- Assumptions: 현재 전제
- Evidence: 확인한 파일, 명령, 화면, 데이터
- Failure Search: 일부러 찾아본 실패 가능성
- Decision: 진행, 수정, escalation 중 무엇을 선택했는가
- Residual Risk: 아직 확인하지 못한 부분
