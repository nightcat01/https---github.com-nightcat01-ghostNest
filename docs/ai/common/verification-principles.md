# Verification Principles

이 문서는 공용 검증 기준이다.

## 기본 원칙

- 변경한 범위에 맞는 가장 작은 검증부터 실행한다.
- normal 이상은 가능하면 타입체크나 빌드를 실행한다.
- UI 변경은 페이지 응답 또는 브라우저 확인을 추가한다.
- 실패 로그는 핵심 원인 중심으로 요약한다.
- 실행하지 못한 검증은 이유를 명시한다.

## 검증 예시

- 타입 변경: typecheck
- 빌드 설정 변경: build
- 서버 라우트 변경: 해당 URL 응답 확인
- UI 변경: 주요 화면 접근, 콘솔 오류, 레이아웃 확인
- 반복 조작 UI 변경: 확대/축소, 재생, 선택 전환, 리로드 후 상태 확인
- 저장 흐름 변경: 저장 후 목록, 미리보기, 리로드 복원 확인
- 저장/삭제 변경: dry-run 또는 대상 경로 확인

## 보고 형식

- 실행한 명령
- 통과한 항목
- 실패한 항목
- 실행하지 못한 항목
- 남은 리스크
