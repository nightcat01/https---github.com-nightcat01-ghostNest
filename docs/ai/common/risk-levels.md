# Risk Levels

이 문서는 프로젝트와 무관하게 사용할 수 있는 공용 위험도 기준이다.

## trivial

영향 범위가 좁고 실패해도 되돌리기 쉬운 작업이다.

- 오타, 문구, 주석 수정
- 단일 문서의 작은 보강
- 명백한 타입 오류 수정
- 인접 코드 흐름을 바꾸지 않는 작은 스타일 수정

기본 역할:

- Worker
- Tester light

## normal

단일 기능, 단일 화면, 단일 설정 범위 안에서 끝나는 작업이다.

- 기존 패턴을 따르는 옵션 추가
- 단일 화면의 작은 기능 보강
- 기존 API 호출 경로 추가
- 문서 구조 추가

기본 역할:

- Planner
- Worker
- Reviewer
- Tester

## risky

공통 구조, 런타임, 저장소, 사용자 흐름, 레이아웃에 영향을 줄 수 있는 작업이다.

- 공통 타입 또는 런타임 동작 변경
- 저장, 삭제, 파일 생성, 자동 반영
- 공통 UI/레이아웃 변경
- 외부 임베드 또는 라우팅 영향
- 기존 데이터 구조 변경

기본 역할:

- Planner
- Worker
- Reviewer
- Tester
- 필요한 특화 역할

## 승격 조건

작업 중 다음 조건이 발견되면 위험도를 올린다.

- 수정 파일 수가 예상보다 늘어남
- 공통 영역 수정이 필요해짐
- 저장/삭제/생성 작업이 포함됨
- UI 흐름이 바뀜
- 기존 사용자의 데이터 또는 설정이 영향받음
- 검증은 통과하지만 실제 결과가 어색함
