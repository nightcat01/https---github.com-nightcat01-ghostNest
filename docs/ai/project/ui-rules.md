# GhostNest UI Rules

이 문서는 GhostNest 전용 UI 안정성 기준이다.

## 핵심 대상

- 캐릭터 위치
- 말풍선과 대사창
- 관리 메뉴
- scene/layer 렌더링
- 개발자 도구 화면
- 모바일 레이아웃

## 말풍선/대사창

- 말풍선은 width, height, max-height 제한을 가진다.
- 메뉴가 말풍선 안에 들어갈 때 메뉴 목록은 내부 스크롤로 제한한다.
- 말풍선 크기 변화가 캐릭터를 위아래로 밀어내지 않아야 한다.
- `floating`, `dialogue-box`, `overlay-bottom` 배치를 각각 확인한다.

## 캐릭터/scene

- 큰 이미지가 layout을 무제한 확장하지 않아야 한다.
- scene은 이미지처럼 선택될 수 있지만 내부적으로는 레이어 그룹이다.
- scene/layer는 위치와 크기 제약 안에서 렌더링되어야 한다.
- 캐릭터나 scene 전환 후 이전 캐릭터의 이미지, 미리보기, 선택 상태가 남지 않아야 한다.
- 확대/축소는 프리뷰 DOM을 불필요하게 재생성하지 않아야 하며, 높이 접힘이나 깜빡임을 만들지 않아야 한다.

## 캐릭터 설정 도구

- 각 화면은 사용자가 무엇을 만드는지, 어떤 순서로 진행하는지, 어디에서 완료하는지 보여야 한다.
- 저장, 삭제, 다음 단계 이동은 사용자가 마지막으로 조작하는 영역 근처에 둔다.
- 긴 화면에서는 완료 액션을 찾기 위해 사용자가 상단으로 되돌아가지 않아도 되어야 한다.
- base, parts, scene 선택 UI는 scene이 이미지처럼 보이더라도 이미지 파일과 다른 그룹 데이터임을 혼동시키지 않아야 한다.
- 신규 생성, 저장, 삭제, 캐릭터 전환, 리로드 후 목록과 미리보기가 같은 데이터를 가리켜야 한다.
- 설명 영역은 액션 버튼에 밀려 좁아지지 않아야 하며, 반복되는 저장/삭제 UI는 공통 스타일을 우선 사용한다.

## 메뉴

- 사용자 메뉴와 개발자 메뉴를 구분한다.
- 무거운 메뉴는 panel 표시를 우선 고려한다.
- 말풍선 메뉴는 간단한 선택과 짧은 행동에 적합하다.

## 모바일

- 버튼 텍스트가 터지지 않아야 한다.
- 캐릭터와 말풍선이 화면 밖으로 밀리지 않아야 한다.
- 내부 스크롤과 페이지 스크롤이 충돌하지 않아야 한다.
