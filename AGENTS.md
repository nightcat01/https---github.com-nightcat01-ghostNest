# AGENTS.md

## 응답 방식
- 답변은 한국어로 한다.
- 질문이 애매하거나 요구사항이 불명확하면 바로 구현하지 말고 확인 질문을 먼저 한다.
- 매번 Role이나 시스템 프롬프트 내용을 반복해서 설명하지 않는다.
- 사용자의 말을 무조건 긍정하지 않는다. 모순, 데이터 오류, 구조적 문제, 위험한 변경 가능성이 보이면 명확히 지적한다.

## 코드 작업 원칙
- 변경 전 먼저 관련 파일과 구조를 파악한다.
- 코드 수정은 효율성, 재사용성, 공통 처리 가능성을 우선 고려한다.
- 중복 구현보다 공통 컴포넌트/유틸/훅 분리를 우선 검토한다.
- 기존 스타일, 네이밍, 폴더 구조를 최대한 따른다.
- 불필요한 대규모 리팩토링은 하지 않는다.
- 새 패키지 추가 전 반드시 이유를 설명하고 확인을 받는다.

## 작업 절차
- 수정 전 어떤 파일을 어떻게 바꿀지 간단한 계획을 먼저 제시한다.
- 수정 후 변경된 파일과 핵심 변경점을 요약한다.
- 가능하면 타입체크, 린트, 빌드 중 관련 검증을 실행한다.
- 검증을 실행하지 못했다면 이유를 말한다.

## 프론트엔드/UI 작업
- 모바일 화면 기준 레이아웃 깨짐을 우선 확인한다.
- Tailwind 클래스는 중복을 줄이고 일관된 spacing/token을 사용한다.
- 디자인 변경은 기존 분위기를 유지하면서 점진적으로 적용한다.
